# نظام رصد بيانات وملاحظات الفحص السعودي — EBDA EDU

نظام ويب لتسجيل ومتابعة بيانات المتقدمين للفحص السعودي، مبني على **Google Apps Script** مع **Google Sheets** كقاعدة بيانات.

🔗 **[فتح النظام](https://script.google.com/macros/s/AKfycbwchUBSk2AMqbfOqMezGk1QNgkdeK8-EcnJMcTaW_PGE-mlT8bg_MgBw7zlGs0P-nyLnA/exec)**

---

## المميزات

- **إدارة البيانات المرجعية**: مراكز التدريب، المهن، المقيّمون (مع ربط كل مقيّم بالمهن التي يقيّمها)
- **تسجيل المتقدمين**: سجل فردي لكل متقدم بالنتيجة والملاحظات
- **داشبورد إحصائي**: نسب النجاح والرسوب، رسوم بيانية حسب المهنة والمركز والاتجاه الشهري
- **تقارير مخصصة**: بأي فترة وأي مجموعة معايير
- **سجل المتقدمين**: فلترة متعددة المعايير + بحث حر + تصدير إلى **Excel / Word / PDF**
- **صلاحيات المستخدمين**: مدير (Admin) / إدخال بيانات (Editor) / مشاهدة فقط (Viewer)
- **أداء محسّن**: تخزين مؤقت للبيانات المرجعية والإحصائيات، وتقليل عدد الطلبات إلى الخادم

## بنية المشروع

```
├── src/
│   ├── Code.gs           # الواجهة الخلفية (Apps Script)
│   ├── Index.html        # الواجهة الأمامية
│   └── appsscript.json   # إعدادات المشروع
├── docs/
│   └── index.html        # صفحة الهبوط (GitHub Pages)
└── README.md
```

---

## طريقة التركيب

1. افتح [script.google.com](https://script.google.com) وأنشئ مشروعًا جديدًا.
2. الصق محتوى `src/Code.gs` في ملف الكود.
3. أنشئ ملف HTML باسم **Index** والصق فيه محتوى `src/Index.html`.
4. من إعدادات المشروع فعّل إظهار `appsscript.json` وعدّله ليطابق `src/appsscript.json`.
5. عدّل قيمة `TARGET_FOLDER_ID` في أعلى `Code.gs` لتشير إلى مجلد Drive الذي تريد حفظ قاعدة البيانات بداخله.
6. شغّل الدالة `setup` مرة واحدة لإنشاء الشيتات وحساب المدير.
7. انشر التطبيق: **Deploy → New deployment → Web app**.

> بعد أول دخول، غيّر كلمة مرور المدير فورًا من تبويب **المستخدمون**.

---

## تحديث النسخة المنشورة

بعد أي تعديل على الكود:

**Deploy → Manage deployments → أيقونة القلم → Version: New version → Deploy**

اختيار "New deployment" ينشئ رابطًا جديدًا مختلفًا، وهو غالبًا ليس ما تريده.

## المزامنة التلقائية مع GitHub (اختياري)

بدل النسخ واللصق اليدوي، يمكن استخدام [clasp](https://github.com/google/clasp):

```bash
npm install -g @google/clasp
clasp login
clasp clone <SCRIPT_ID>     # أو clasp create
clasp push                  # رفع التعديلات إلى Apps Script
clasp pull                  # سحب التعديلات منه
```

بهذه الطريقة يصبح المستودع هو المصدر الرسمي للكود.

## ملاحظات أمنية

- لا تضع أي كلمات مرور أو معرّفات حساسة داخل المستودع.
- راجع إعداد صلاحية الوصول للتطبيق (`webapp.access`) قبل نشر الرابط علنًا.
- كلمات المرور مخزّنة كبصمة SHA-256 بدون salt؛ للاستخدام في بيئة إنتاج حقيقية يُفضّل الاعتماد على تسجيل دخول Google بدلًا من كلمات مرور مخزّنة يدويًا.
